# DATA_SCHEMA

Описание структуры данных каталога монет.

## Основной файл

Основной файл каталога — `coins.json`.

Минимальная структура:

```json
{
  "version": 2,
  "updatedAt": "2026-06-28T00:00:00.000Z",
  "series": [],
  "coins": []
}
```

## Корневые поля

| Поле | Тип | Обязательное | Описание |
|---|---:|---:|---|
| `version` | number | да | Версия структуры каталога. Сейчас используется `2`. |
| `updatedAt` | string | да | ISO-дата последнего обновления каталога. |
| `series` | array | да | Справочник серий внутри конкретного каталога. |
| `coins` | array | да | Список монет. |

## Серии

Серии хранятся в `coins.json`, потому что это пользовательская часть каталога.

```json
{
  "id": "series_gdl_abc123",
  "name": "Монеты ВКЛ"
}
```

У монеты серии задаются через массив `seriesIds`:

```json
{
  "id": "coin_001",
  "seriesIds": ["series_gdl_abc123"]
}
```

## Эмитенты

Справочник эмитентов хранится отдельно:

```text
data/issuers.json
```

Новые и отредактированные монеты должны хранить эмитента через `issuerId`:

```json
{
  "issuerId": 2
}
```

Для обратной совместимости поддерживаются старые варианты:

```json
{
  "country": "ВКЛ"
}
```

```json
{
  "countryId": 2
}
```

```json
{
  "country": "2"
}
```

Если у монеты есть `issuerId`, поле `country` при компактном сохранении не записывается.

## Поля монеты

Единственное обязательное поле монеты — `id`.

Все остальные поля необязательные и не записываются в JSON, если они пустые.

### Основное

| Поле | Тип | Описание |
|---|---:|---|
| `id` | string | Уникальный ID монеты. |
| `issuerId` | number | ID эмитента из `data/issuers.json`. |
| `country` | string | Legacy-поле для старых записей. |
| `nominal` | string | Номинал. |
| `title` | string | Название. |
| `year` | string | Год или диапазон лет. |
| `mint` | string | Монетный двор. |
| `strikeType` | string | Тип чеканки. |
| `status` | string | Статус монеты. По умолчанию не записывается и считается `in_collection`. |

### Статус

Поддерживаются два статуса:

```text
in_collection
sold
```

Если `status` отсутствует, монета считается находящейся в коллекции.

Старые значения вроде `В коллекции` и `Продано` нормализуются автоматически.

### Продажа

| Поле | Тип | Описание |
|---|---:|---|
| `salePrice` | string | Сумма продажи. Записывается только если `status = sold`. |

Спрэд не хранится в JSON. Он вычисляется на экране как:

```text
salePrice - purchasePrice
```

если обе суммы заполнены.

### Характеристики

| Поле | Тип | Описание |
|---|---:|---|
| `material` | string | Материал. |
| `fineness` | string | Проба. |
| `weight` | string | Вес. |
| `diameter` | string | Диаметр. |
| `thickness` | string | Толщина. |
| `mintage` | string | Тираж. |
| `condition` | string | Состояние. |

### Покупка и оценка

| Поле | Тип | Описание |
|---|---:|---|
| `purchaseDate` | string | Дата приобретения. Обычно `YYYY-MM-DD`. |
| `purchasePrice` | string | Цена покупки в EUR. Символ валюты не хранится. |
| `source` | string | Источник покупки. |
| `currentValue` | string | Текущая оценка в EUR. Символ валюты не хранится. |

Цены хранятся текстом, но нормализуются как десятичные значения. Запятая приводится к точке, символы валюты удаляются при сохранении.

### Каталог

| Поле | Тип | Описание |
|---|---:|---|
| `catalogNumber` | string | Каталожный номер. |

### Грейдинг

| Поле | Тип | Описание |
|---|---:|---|
| `slabCompany` | string | Грейдинговая компания. |
| `slabNumber` | string | Номер слаба. |
| `slabUrl` | string | Ссылка на страницу монеты у грейдера. |

`slabUrl` должен быть валидной ссылкой `http://` или `https://`. Если пользователь вводит домен без протокола, приложение пытается нормализовать его в `https://...`.

### Фото

Фото хранятся внутри объекта `photos`:

```json
{
  "photos": {
    "obverse": "images/coin_001_obverse.jpg",
    "reverse": "images/coin_001_reverse.jpg"
  }
}
```

| Поле | Тип | Описание |
|---|---:|---|
| `photos.obverse` | string | Относительный путь к фото аверса. |
| `photos.reverse` | string | Относительный путь к фото реверса. |

В форме пустые поля фото заполняются префиксом `images/` для удобства, но bare-значение `images/` не сохраняется в JSON.

### Дополнительно

| Поле | Тип | Описание |
|---|---:|---|
| `comment` | string | Комментарий. Поддерживает переносы строк. |
| `storageLocation` | string | Legacy-поле. В UI не используется, но старые данные не ломает. |

## Компактное сохранение

При сохранении каталог проходит через `CoinDB.compactCatalog()`.

Правила:

- пустые необязательные поля не записываются;
- `status = in_collection` не записывается;
- `salePrice` записывается только для проданных монет;
- `country` не записывается, если есть `issuerId`;
- `photos` не записывается, если оба пути пустые;
- `seriesIds` не записывается, если список пустой.

## Внутренние кэши приложения

Кэши не являются частью `coins.json`.

Используются:

- `localStorage` — рабочая копия каталога и настройки UI;
- IndexedDB — file/directory handles, drafts, persistent image cache;
- Cache Storage — PWA app shell через service worker.

Persistent image cache хранит копии успешно прочитанных фото и используется как fallback на Android, если доступ к внешней папке пропадает.
