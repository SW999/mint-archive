# PROJECT_STATE

Актуальное состояние проекта локальной PWA для каталога монет.

## Назначение

Приложение — приватный offline-first каталог монет для Windows и Android. Основной сценарий: пользователь открывает локальную папку каталога, где находятся `coins.json` и папка `images/` с фото монет.

## Технологии и ограничения

- Чистые `HTML/CSS/JS`.
- Без React, сборки, npm-зависимостей и локального сервера.
- PWA с `service-worker.js`.
- Данные хранятся локально.
- Основной файл данных: `coins.json`.
- Справочник эмитентов: `data/issuers.json`.
- Фото монет: относительные пути в `coins.json`, обычно `images/...`.
- Не трогать `images/logo.png`: это основной знак в шапке приложения.

## Текущая архитектура

Приложение переведено на SPA внутри `index.html`.

Актуальные маршруты:

```text
#/list
#/coin/<id>
#/edit/<id>
#/new
#/stats
```

Отдельные страницы `coin.html`, `form.html`, `stats.html` удалены из актуальной структуры. В `service-worker.js` оставлены редиректы со старых URL на SPA-маршруты для совместимости:

```text
coin.html?id=... -> index.html#/coin/...
form.html?id=... -> index.html#/edit/...
form.html -> index.html#/new
stats.html -> index.html#/stats
```

## Хранение и сохранение данных

- Каталог хранится в `localStorage` как рабочая копия.
- При открытии папки используется File System Access API.
- При сохранении приложение пытается перезаписать выбранный `coins.json`.
- Если прямое сохранение недоступно, используется скачивание JSON.
- JSON всегда сериализуется через `JSON.stringify` внутри `CoinDB.stringifyCatalog()`.
- Пустые необязательные поля не записываются в JSON.
- При сохранении через открытую папку создается автоматический backup.
- Backup ограничен последними 5 файлами `coins_backup_*.json`.

## Фото и Android

На Android 13 выявлена нестабильность доступа к внешней папке через File System Access API: часть фото может перестать читаться с ошибкой доступа.

Текущее решение:

- фото закрепляются лениво во внутреннем IndexedDB-кэше приложения;
- массовая вычитка всех фото не используется;
- если фото успешно прочитано из внешней папки, оно сохраняется во внутренний image cache;
- при следующем отображении приложение сначала пробует использовать внутренний кэш;
- если доступ к папке пропал, приложение может показать ранее закрепленную копию;
- лимит persistent image cache: 150 фото;
- есть отдельная кнопка `Очистить кэш фото`, она не трогает `coins.json`, backup и оригинальные файлы.

## Service worker

Актуальная версия cache name после этого этапа:

```text
coins-pwa-v25
```

В app shell кэшируются основные JS/CSS, `index.html`, `manifest.json`, `data/issuers.json`, PWA-иконки, `images/logo.png` и placeholder-изображения.

`data/issuers.json` загружается по network-first стратегии, чтобы ручные изменения справочника быстрее подтягивались.

## Текущие реализованные функции

- SPA-список монет.
- SPA-карточка монеты.
- SPA-добавление/редактирование.
- SPA-статистика.
- Фильтры по эмитенту, материалу, статусу, состоянию, серии.
- Переключение карточки/таблица.
- Сумма выбранных монет без проданных.
- Статусы: `in_collection`, `sold`.
- Проданные монеты имеют красный тег и исключены из сумм покупок.
- Поля продажи: `salePrice`, спрэд вычисляется на экране и не хранится.
- Серии на уровне каталога и `seriesIds` у монет.
- Справочник эмитентов через `issuerId` + совместимость со старым `country`.
- Грейдинг: `slabCompany`, `slabNumber`, `slabUrl`.
- Ссылка на грейдера открывается в новой вкладке.
- Статистика: сумма покупок, медианная цена, эмитенты, материалы, состояние, века, периоды.
- Черновики формы в IndexedDB.
- Автоматические backup-файлы.
- Persistent image cache в IndexedDB.

## Правила дальнейшей разработки

- Патчи делать маленькими логическими этапами.
- В ZIP включать только измененные файлы + `PATCH_NOTES.md`.
- Если нужны удаления файлов, явно указывать их отдельно.
- Перед выдачей патча запускать `node --check` для всех JS.
- Не накапливать гарантированно неиспользуемый код.
- Не дублировать CSS-селекторы без необходимости.
- Не менять `images/logo.png` и шапку без явной просьбы.
- Не добавлять React, сборку, зависимости или локальный сервер.
